### 1.3 Risk Assessment

**Risk Level:** MEDIUM-HIGH

**Primary Risks:**
1. **E-E-A-T Compliance Risk (HIGH)**
   - YMYL medical content requires verified doctor credentials
   - Incorrect attribution could result in Google penalties
   - Mitigation: Every page includes doctor credentials, medical disclaimers, review process

2. **Content Quality Risk (MEDIUM)**
   - Low-quality content hurts rankings
   - Duplicate content penalties
   - Mitigation: Original 1,500+ word content per page, doctor review required

3. **Technical Implementation Risk (MEDIUM)**
   - Schema markup errors cause rich result failures
   - Performance regression impacts rankings
   - Mitigation: Schema validation, Lighthouse audits, staging environment testing

4. **Timeline Risk (MEDIUM)**
   - 16-week implementation could delay ROI
   - Resource constraints could extend timeline
   - Mitigation: Phased rollout, parallel development, clear milestones

5. **HIPAA Compliance Risk (LOW)**
   - Patient testimonials/photos must be HIPAA-compliant
   - Mitigation: Signed release forms, PHI redaction, compliance review

**Risk Mitigation Strategy:**
- Phase-by-phase validation before proceeding
- Weekly quality reviews
- Compliance checklist for every page
- Rollback procedures documented

### 4.2 Next.js Implementation Patterns

#### 4.2.1 Service Page Template Structure
```typescript
// app/services/[service-slug]/page.tsx
import { Metadata } from 'next';
import { generateServiceSchema } from '@/lib/schema';

interface ServicePageProps {
  params: { 'service-slug': string };
}

export async function generateMetadata({ params }: ServicePageProps): Promise<Metadata> {
  const service = getServiceData(params['service-slug']);

  return {
    title: `${service.name} in Orange County | EyeCare Center OC`,
    description: service.metaDescription,
    keywords: service.keywords,
    openGraph: {
      title: `${service.name} | EyeCare Center OC`,
      description: service.metaDescription,
      images: [{ url: service.ogImage }],
    },
  };
}

export default function ServicePage({ params }: ServicePageProps) {
  const service = getServiceData(params['service-slug']);

  return (
    <>
      {/* Schema markup */}
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{
          __html: JSON.stringify(generateServiceSchema(service)),
        }}
      />

      {/* Page content */}
      <ServicePageLayout service={service} />
    </>
  );
}
```

### 4.3 Performance Requirements

**Core Web Vitals Targets:**
- LCP (Largest Contentful Paint): ≤2.5 seconds
- INP (Interaction to Next Paint): <200ms
- CLS (Cumulative Layout Shift): <0.1

**Lighthouse Scores:**
- Performance: 85+ (mobile), 90+ (desktop)
- Accessibility: 95+
- Best Practices: 90+
- SEO: 95+

**Image Optimization:**
- Format: AVIF primary, WebP fallback, PNG/JPEG for compatibility
- Compression: Target <200KB per image
- Lazy loading: Below-fold images
- Priority loading: Hero images, above-fold content
- Dimensions: Always specify width/height to prevent CLS

**Code Splitting:**
- Route-based code splitting (Next.js automatic)
